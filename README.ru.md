## Описание проекта

Этот проект представляет собой прототип простейшего UI-фреймворка, написанного на Go с использованием библиотеки **raylib-go** (`github.com/gen2brain/raylib-go/raylib`). Основная цель проекта — исследование и реализация базовых принципов построения пользовательского интерфейса, включая работу с визуальными элементами, слоями, анимациями и пользовательским вводом, а также выстраивание корректной архитектуры приложения.

Проект не является полноценным UI-движком, но служит учебной и экспериментальной базой для понимания того, как можно организовать UI-систему с нуля.

## Основные возможности

* Использование **raylib-go** для отрисовки и обработки ввода
* Создание и отображение UI-элементов на основе `Rectangle`
* Поддержка анимаций UI-элементов
* Работа со слоями (layer system):

  * управление порядком отрисовки
  * корректное перекрытие элементов
* Взаимодействие с элементами UI:

  * hover / unhover
  * left / right click
  * drag & drop
* Корректная обработка пользовательского ввода (мышь)
* Динамическое добавление и удаление элементов
* Архитектура, ориентированная на расширяемость и поддержку

## Архитектура

Проект построен вокруг центральной сущности `Application`, которая отвечает за:

* жизненный цикл приложения (init / update / render / close)
* хранение и управление UI-элементами
* обработку пользовательского ввода
* управление активным и наведённым элементом

UI-элементы реализуют общий интерфейс (`Element`), который включает:

* `Draw`
* `Update`
* обработку взаимодействий (hover, click, drag, drop)
* работу со слоями (`GetLayer`)
* определение границ (`GetBounds`)

## Работа со слоями

Каждый UI-элемент имеет свой `Layer`. Элементы сортируются по возрастанию слоя, благодаря чему элементы с более высоким `Layer` рисуются последними и визуально располагаются поверх остальных.

## Цели проекта

* Изучение и практическое применение **raylib-go**
* Понимание принципов построения UI-систем с нуля
* Реализация корректной обработки пользовательского ввода
* Работа с UI-элементами без привязки к индексам
* Проектирование чистой и расширяемой архитектуры
* Подготовка основы для будущего развития (кнопки, контейнеры, layout-системы)
